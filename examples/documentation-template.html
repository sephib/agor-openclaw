API Documentation

Welcome to the TaskFlow API documentation. This guide will help you integrate TaskFlow into your applications and automate your workflow.

API Version: v2.0
Base URL: https://api.taskflow.com/v2

Getting Started

To use the TaskFlow API, you'll need to authenticate your requests using an API key. Follow these steps to get started:

  1. Sign up for a TaskFlow account
  2. Navigate to Settings → API Keys
  3. Generate a new API key
  4. Include the key in your request headers

Authentication

All API requests must include your API key in the authorization header:

Authorization: Bearer YOUR_API_KEY
Security Note: Never expose your API key in client-side code or public repositories. Keep your keys secure and rotate them regularly.

Making Your First Request

Here's a simple example to fetch all projects in your workspace:

curl https://api.taskflow.com/v2/projects \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

Response Format

All API responses are returned in JSON format:

{
  "data": [
    {
      "id": "proj_123abc",
      "name": "Website Redesign",
      "status": "active",
      "created_at": "2024-01-15T10:30:00Z"
    }
  ],
  "meta": {
    "total": 1,
    "page": 1,
    "per_page": 20
  }
}

Common Operations

Create a Task

Create a new task in a project:

POST /v2/projects/:project_id/tasks

{
  "title": "Design homepage mockup",
  "description": "Create initial design concepts",
  "assignee_id": "user_456def",
  "due_date": "2024-02-01",
  "priority": "high"
}

Parameters

Parameter Type Required Description
title string Yes Task title (max 200 characters)
description string No Detailed task description
assignee_id string No User ID to assign the task
due_date date No Task due date (ISO 8601 format)
priority enum No Priority level: low, medium, high

Rate Limits

The API enforces rate limits to ensure fair usage:

  • Free tier: 100 requests per hour
  • Professional: 1,000 requests per hour
  • Enterprise: Custom limits

Rate limit information is included in response headers:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 998
X-RateLimit-Reset: 1609459200

Error Handling

The API uses standard HTTP status codes to indicate success or failure:

Status Code Meaning Description
200 OK Request successful
201 Created Resource created successfully
400 Bad Request Invalid request parameters
401 Unauthorized Invalid or missing API key
404 Not Found Resource not found
429 Too Many Requests Rate limit exceeded
500 Server Error Internal server error
