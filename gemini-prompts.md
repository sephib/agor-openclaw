## Initial Generation Prompts

These are your starting point prompts for creating complete websites.

### Universal Website Generator

```
Create a complete, production-ready single-page HTML website with inline CSS.

Content:
[Your website content here - sections, text, links, etc.]

Design Requirements:
- Modern, professional design
- Fully responsive (mobile, tablet, desktop)
- Clean typography and layout
- [Color scheme - e.g., "Blue and white" or specific hex codes]
- Smooth scrolling navigation
- Accessibility features (proper heading structure, alt tags)

Technical Requirements:
- Single HTML file
- All CSS inline in <style> tag
- No external dependencies
- Cross-browser compatible
- Include placeholder comments for images

Generate the complete HTML code now.
```

**When to use:** Starting any new website project
**Expected result:** Complete, working HTML file

---

### Minimal Starter Template

```
Create a minimal, single-page HTML website with clean design.

Include:
- Header with navigation
- Hero section with heading and call-to-action
- 2-3 content sections
- Contact/footer section

Keep the design simple and professional. Use a neutral color palette.
All code should be in one HTML file with inline CSS.
```

**When to use:** When you want a basic foundation to build upon
**Expected result:** Minimal but complete starter template

## Responsive Design Prompts

### Fix Mobile Issues

```
The [specific element] doesn't work well on mobile. Fix by:
- Making it stack vertically on screens below 768px
- Increasing touch target sizes for buttons (minimum 44px)
- Hiding [element] on mobile and showing [alternative]
- Adjusting font sizes for mobile (reduce by 20%)
```

---

### Improve Mobile Navigation

```
Convert the navigation menu to a mobile-friendly hamburger menu:
- Show hamburger icon on screens below 768px
- Menu slides in from the side when clicked
- Include smooth animations
- Keep desktop horizontal menu unchanged
```

---

### Tablet Optimization

```
Optimize the layout for tablet (768px - 1024px):
- Change the [section] from 4 columns to 2 columns
- Adjust image sizes to maintain aspect ratio
- Increase padding slightly from mobile
- Make sure touch targets are at least 44px
```

---

### Desktop Enhancements

```
Add desktop-specific enhancements for screens above 1200px:
- Wider content container (1400px max-width)
- Larger font sizes (20% increase)
- Show additional visual elements hidden on mobile
- Add hover effects that don't work on touch devices
```

## Prompt Patterns & Tips

### The "Build Then Refine" Pattern

1. **Initial Build:**
   ```
   Create a [type] website with [basic requirements]
   ```

2. **Visual Refinement:**
   ```
   Make it more visually appealing with [specific design changes]
   ```

3. **Content Updates:**
   ```
   Update the content in [section] to [new content]
   ```

4. **Mobile Polish:**
   ```
   Fix mobile responsiveness issues in [section]
   ```

---

### The "Specific Context" Pattern

**Instead of:**
```
Make the buttons bigger
```

**Use:**
```
Increase the call-to-action button size from 1rem to 1.25rem padding
and make the font size 1.125rem for better visibility
```

**Why:** Specific measurements get exact results

---

### The "Show Don't Tell" Pattern

**Instead of:**
```
Make it look more modern
```

**Use:**
```
Add these modern design elements:
- Rounded corners (8px border radius)
- Subtle shadows (0 4px 6px rgba(0,0,0,0.1))
- Smooth transitions (0.3s ease)
- Gradient accents
- Generous whitespace
```

**Why:** Specific examples eliminate ambiguity

---

### The "Reference Comparison" Pattern

```
Make the [section] look similar to the hero section in terms of:
- Color scheme
- Typography
- Spacing
- Visual style

But with [specific differences]
```

**Why:** Leverages existing good design as a template

---

### The "Problem-Solution" Pattern

```
Problem: The [element] is [specific issue]

Fix it by:
1. [Specific change]
2. [Specific change]
3. [Specific change]

Test by [how to verify the fix]
```

**Why:** Clear problem statement ensures the fix addresses the real issue

## Best Practices

### ✅ DO:

- **Be specific** about colors (use hex codes, not "blue")
- **Provide context** about the purpose and audience
- **Include dimensions** when requesting layout changes
- **Specify breakpoints** for responsive requests (e.g., "below 768px")
- **Reference existing sections** for consistency
- **Ask for explanations** when learning new techniques
- **Test mobile** separately and explicitly

### ❌ DON'T:

- Use vague terms like "better" or "nicer"
- Assume Gemini knows your brand guidelines
- Request everything at once without structure
- Forget to specify "inline CSS" if you need a single file
- Skip testing on actual mobile devices
- Request features that need backend code (use APIs instead)
